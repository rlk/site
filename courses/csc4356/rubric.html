CSC 4356 — Interactive Computer Graphics

Programming Project Scoring Rubric

Criteria Inadequate Needs Improvement Meets Expectations Exceeds Expectations
Protocol
10%
Was the project submitted for grading correctly and on time?
  • Program is submitted incorrectly or late.
  • Program is submitted correctly and on-time.
Syntax
10%
Is the code formatted in a way that best communicates its function?
  • Code does not compile or cannot be interpreted.
  • Indentation is inconsistent or does not convey block structure.
  • No explanatory comments.
  • Indentation is consistent but has a mixture of tabs and spaces.
  • Lengthy regions of blank lines.
  • Incomplete explanatory comments.
  • Indentation consistently conveys block structure.
  • Indentation consistently uses all tabs or all spaces.
  • Each function or object is documented with a comment as to its purpose.
  • In-line commentary elucidates complex code.
Structure
20%
Is the code structured efficiently and effectively? Is it good programming?
  • Significant quantities of dead or useless code.
  • Redundant code.
  • Heavy use of global variables.
  • Improper use of member variables in lieu of argument passing.
  • Significant quantities of commented-out code.
  • Unnecessary code.
  • Lengthy functions that perform many tasks.
  • Each function performs one well-defined task.
  • Each line of code performs a necessary operation.
  • Code structure represents a well-formed, well-reasoned style unique to the student and does not parrot examples given in class.
Resource Management
20%
Does the code utilize resources optimally?
  • Resources are acquired and lost without release (leaked). This is especially egregious in loop bodies such as the render loop.
  • Tasks are performed in the wrong place, such as initialization tasks performed in the render loop.
  • Unnecessary repetition of resource acquisition with release (unleaked).
  • Program uses resources efficiently and allows them to release automatically upon exit.
  • All acquired resources are released prior to exit.
Project Objectives
40%
Does the code correctly exercise the key concepts of the project?
  • Code demonstrates lack of understanding of key concepts.
  • Program does not run or does not produce useful output.
  • Code demonstrates misunderstanding or incomplete understanding of key concepts.
  • Program executes and produces some correct output.
  • Code correctly exercises all key concepts.
  • Program executes and produces all correct output.
  • Program generalizes the required objective effectively or explores additional related functionality in a fruitful manner.
