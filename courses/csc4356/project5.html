Project 5 — Scene Management

Yet again we extend our previous project. We remain focused on loading and rendering 3D mesh data, but this time we'll be dealing with multiple meshes simultaneously. All of our previous projects have assumed there is only one mesh, which simplifies everything. With this project, we'll do familiar things, but we'll need to be more careful about how we do them.

Here is a ZIP file containing several textured meshes. There are 5 different palm trees, 3 different plants, and 1 desert background. Our objective is to place all 9 of these together in a scene, rendering with per-fragment lighting, as before. A number of new requirements arise from this:

  • We'll need an independent vertex buffer, element buffer, and texture object for each mesh. This must be implemented using a structure or object to encapsulate and separate mesh data. Do not simply copy and paste the mesh loading and drawing code 9 times.
  • Each of our previous projects placed the object at the center of the world. However, if all of our plants and trees are placed at the center, then they will all appear on top of one another. Instead, use the ModelView matrix to translate them to different positions.
  • Our old tumble interaction is useful for examining simple objects, but it's not appropriate for wandering around a desert oasis. Replace the tumble interaction with a look interaction, allowing the user to look around with the mouse while walking in any direction with the keyboard.
  • These plants and trees have semi-transparent textures. Transparency interferes with depth testing, so the fragment shader must account for this.

The mesh data format is the same as before, with one addition. Because each DAT mesh file may have a different TGA texture image, the DAT must indicate which TGA it requires. Previously, the TGA file name was hardcoded, but now the DAT file gives the name of the TGA image file on the first line, following the number of vertices and triangles. For example, here is the first line of fern.dat:

2460 3520 fern_diffuse.tga

Submit your source code in a file named project7.cpp. Submit your shaders as vshader7.glsl and fshader7.glsl. Submit a screenshot depicting a representative view of the scene in a file name screenshot7.png. As usual, be careful about formatting, and please remove any dead code from your project source.

If your computer completely chokes trying to render all 9 meshes, then include at least 3, one of which is the desert background.

(You may note there are several extra TGA files in the ZIP. The use of these is not required, but it's interesting to think about what one might do with them.)
