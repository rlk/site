Project 2 — How to Triforce

Overview

The purpose of Project 2 is to begin to use WebGL to display geometry. This entails defining a simple geometric figure, conveying that definition to GPU memory, and triggering the rendering of the figure by the GPU.

This is accomplished with code very similar to an example given in Chapter 3. However, that example must be expanded upon slightly. This expansion requires an understanding of some WebGL basics.

Specifics

  • Continue reading the WebGL Programming Guide, into Chapter 3, at least to page 91. This chapter lays out the procedures for WebGL geometry specification in detail.

  • Following the instructions given in Using Git with classes.csc.lsu.edu, create a new remote repository called "project2", and clone it to your local computer.

  • As before, you'll need to include these files in your project:

    • webgl-utils.js
    • webgl-debug.js
    • cuon-utils.js

  • Type in the HelloTriangle example of Listing 3.3. This example, as printed, is incomplete. You'll need to fill in the gaps in that code using the understanding gained from Project 1 and the reading of Chapters 2 and 3. You'll have an HTML file and a Javascript file. Name them:

    • HelloTriangle.html
    • HelloTriangle.js

    Format your code consistently, especially when it comes to indentation and brace placement. This is a critical part of good programming. When in doubt, follow Crockford's conventions. Your text editor probably formats code automatically, so there's no excuse for bad formatting. We're going to deduct points for bad format.
