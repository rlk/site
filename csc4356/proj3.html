Project 3: Per-pixel Illumination

The purpose of this project is to gain experience using GLSL to perform high quality illumination.

1. Scene

As with Project 2, create a scene composed of at least two objects that you have created or downloaded, and make it interactively navigable. The OBJ module is at your service. One of these objects must be the “Thirteen Box” (OBJ, MTL, diffuse texture, normal texture, specular texture).

2. Material

What distinquishes this from Project 2 is this: Your models must have texture-based materials consisting of at least two texture images. Most likely, these two textures will be a diffuse color map and a normal map, but you may be creative. Note that the required Thirteen Box meets this criterion.

3. Illumination

Complicating this ever so slightly, you must implement a GLSL spotlight using a projective texture. Let the spotlight be interactive in some fashion, perhaps like the viewing-point-1 example. Allow the user to select from between at least two spotlight textures.

4. Application Code

Enable the use of Tangent vectors in your application code. The OBJ Module will compute the tangent vectors for any textured model, but you'll need to make a point of linking your shaders in such a way that the OBJ module's use of vertex buffer objects is correctly mapped onto your shader's user-defined vertex attribute.

5. GLSL

Render the Thirteen Box correctly using tangent space normal mapping and diffuse and specular texturing. You've seen this done in class. Apply the same or similar per-pixel shading to your other objects.

6. Extra

As always, you may do whatever you like to meet your lagniappe requirement. Consider layered textures to add dirt and grime to an already-textured object. Try a procedural texture to enhance the detail of existing image textures. Cubic environment mapping would be great, and using the per-pixel normal to perform cubic environment reflection or refraction would be super-great.

6. Document

Once again, make a web site documenting your work completely. Include screenshots of your scene and materials. Include all source code and assets necessary to reproduce the project in a ZIP archive. Describe everything you've done and explain how it works in detail. Use your documentation to convince your instructor that you understand what you've presented instead of having simply copied it from a web site.
